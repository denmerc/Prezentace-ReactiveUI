Reactive UI - představení

  • využívá Reactive extensions
  • MVVM Framework by Paul Betts http://reactiveui.net
  • přepracovaná verze 5.2 pro .NET 4.5 => Win8, WinRT, WP8, iOS, Android
  • souběžný vývoj větve verze 4 pro .NET 4.0 a tím i podpora třeba pro Silverlight
  • NuGet - Install-Package ReactiveUI
  • using ReactiveUI;

MVVM

  • hojně používaný prezentační vzor zejména v MS
  • přebírají jej i ostatní platformy
    • WPF
    • WinForms
    • JavaScript
    • Cocoa
    • Android
  • ViewModel publikuje
    • Property
    • Command

INotifyPropertyChanged

  • v ReactiveUI existuje základní třída ReactiveObject
  • pomocné metody a třídy
    • RaiseAndSetIfChanged
    • ObservableForProperty
    • ObservableAsPropertyHelper
    • RxApp.GetFieldNameForPropertyNameFunc

ICommand

  • v ReactiveUI existuje třída ReactiveCommand
  • dokáže spouštět asynchronní operace a publikovat výsledek do MainThread vlákna
  • automaticky zajišťuje "disablování" prvku na UI změnou vlastnosti CanExecute
  • umožňuje skládání více Commandu do jednoho

Binding v code behind

  • nechte práci na XAMLu designérům
  • nejedná se o deklarativní binding jako třeba v podání Caliburn.Micro = máte vše pod kontrolou
  • implementace IViewFor<TViewModel>
    • Bind
    • OneWayBind
    • BindCommand

MessageBus

  • slouží k decouplingu jednotlivých částí aplikace
  • implementuje rozhraní IMessageBus
  • výchozí implementace ve třídě MessageBus a instance publikovaná přes statickou vlastnost Instance
  • zasílání a přijímání pomocí
    • RegisterMessageSource - můžeme registrovat přímo Command a publikovat tak zprávy
    • SendMessage - odeslání zprávy a kontraktu
    • Listen - přihlášení k odběru událostí a rozlišení dle kontraktu

Routing

  • slouží k navigaci mezi jednotlivými View uvnitř aplikace
  • ViewModel implementuje rozhraní IRoutableViewModel
  • Root window obsahuje RoutedViewHost prvek do kterého se jednotlivá View umísťují
